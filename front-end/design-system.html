📐 Component Specifications

Header / Top Bar

Fixed sticky position, white background with subtle border. Contains app badge, portal title, and profile section.

Height: 64px | Shadow: var(--shadow)

Sidebar (Admin)

Collapsible left navigation panel. Desktop only. Smooth collapse animation.

Width: 260px (expanded), 70px (collapsed) | Radius: 12px

Bottom Navigation (Mobile)

4-slot mobile bottom nav for Driver & Citizen portals. Sticky to bottom.

Height: 60px | Icons + labels | Active indicator

Card Container

Standard reusable card. Used for every content block.

Radius: 16px | Border: 1px solid var(--border) | Shadow: var(--shadow)

KPI Card

Data display card with gradient background and trend indicator.

Background: gradient #ecfdf5 to #f0fdf4 | Radius: 16px

Modal / Bottom Sheet

Mobile: Bottom sheet overlay. Desktop: Centered modal with backdrop.

Backdrop: rgba(0,0,0,0.5) | Radius: 16px top (mobile)

📱 Responsive Breakpoints

Mobile (360px)

Single column layout. Bottom navigation. Full-width cards.

Driver & Citizen optimized for this viewport

Tablet (768px)

2-column grid layouts. Sidebar collapses or hides. Touch-friendly controls.

Intermediate breakpoint with adjusted spacing

Desktop (1440px)

Multi-column grids. Sidebar visible. Full feature set.

Admin portal optimized for desktop experience

⭐ Design System Notes

  • ✓ All text has sufficient contrast for WCAG AA compliance
  • ✓ Buttons have minimum 44px touch target on mobile
  • ✓ Icons from Font Awesome 6.4.0 (400+ icons available)
  • ✓ Spacing uses 0.5rem increments (8px base)
  • ✓ All transitions use 0.3s ease timing
  • ✓ Glassmorphism: backdrop-filter: blur(10px) + rgba backgrounds
  • ✓ Cards use consistent 12-24px border radius
  • ✓ Focus states visible for keyboard navigation
  • ✓ Loading states use spinners + opacity reduction
  • ✓ Error states highlighted with alert colors and icons
